Project Titles

Pinpoint: A Vehicle Tracking and Reservation APP UI/UX Case Study

Project Description

Vehicle tracking apps enable you to monitor the whereabouts of your vehicles at all times.
Additionally, it enables you to track the drivers' route. reserve vehicle and manage schedules as an admin

Designer's Role

  • User Research
  • User Persona
  • User Journey Map
  • Site Map
  • Wireframing
  • UI/UX Design

Project Goal

Make App Role Based and Easy to use in order to track live location and provide easy navigation of all features

Target Audience

  • Students
  • Faculty Members
  • Admin

Key Challenges

  • Make Website Responsive
  • All features are mixed no role based features
  • User Frustration
  • Navigation problems
  • UI is not engaging and pleasing

Research Study Details

Studying user requirements, behavior, and motivation involves using observational techniques, task analyses, and feedback loops. I have used both quantitative and qualitative research approaches.
I spoke with 27 people—students, instructors, and administrators—of all ages and genders to learn about the challenges they have while using the Pinpoint app for car monitoring, reservations, and schedule management.
I also conducted a survey of 55 people, and the majority of them said they were upset by the app's overwhelming amount of options and unclear navigation since it was not role-based.

Project Title

Personal Portfolio: A Mobile App UI/UX Case Study

Project Description

A mobile application where users can show case UX Designs and clients can order designs and pick customized designs for their projects.
User can use app for freelance activities to provide services

Designer's Role

  • User Research
  • User Persona
  • Site Map
  • Wireframing
  • UI/UX Design

Project Goal

Easy to use UX Portfolio app for UX Designer and UX Cleints that need UX related work so that
they can order designs from UX Designers while using this App

Target Audience

  • UX Designers
  • UX Clients
  • Tech Companies

Key Challenges

  • Add to cart and check options not clear
  • Missing Customized Design Catagories
  • Design is not asthetically pleasing

Research Study Details

Research study includes understanding user requirements, behavior, and motivation is accomplished through observation techniques, task analysis, and feedback loops.
I used two research methods: i) Quantitative
and ii) Qualitative
. I interviewed 14 people of all ages and genders, including UX Designers, UX Clients, and people of Tech Companies, to learn about the challenges they have while using the Portfolio app for selling services, buying UX services and Ordering Customized UX Designs.
I also conducted a poll of 105 people, the majority of them stated that they do not find add to cart and customized design catagories functions in the app and are annoyed by too many options and complicated navigations.
